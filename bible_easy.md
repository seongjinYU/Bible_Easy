# 📖 성경 구절 자동완성 유틸리티 개발 계획서 (macOS MVP)

## 1. 프로젝트 개요

macOS 환경에서 백그라운드로 실행되며, 사용자가 카카오톡, 메모장, 웹 브라우저 등 **모든 텍스트 입력 필드**에서 특정 패턴(예: `#창 1:1`)을 입력하면, 이를 감지하여 해당 성경 구절(개역한글)의 전체 텍스트로 자동 변환(치환)해주는 시스템 유틸리티 프로그램입니다.

## 2. 핵심 목표 (MVP)

* **플랫폼:** macOS 전용 버전 우선 개발
* **핵심 기능:**
  * 단일 절: `#창 1:1` → "태초에 하나님이 천지를 창조하시니라 [창세기 1:1]"
  * 범위 조회: `#창 1:1-3` → "1 태초에... 2 땅이... 3 하나님이... [창세기 1:1-3]"
* **데이터:** 로컬 DB(SQLite)를 내장하여 오프라인에서도 작동
* **배포:** 지인에게 배포 가능한 `.app` 또는 `.dmg` 파일 생성

## 3. 필요 기술 (Tech Stack)

* **프레임워크: Tauri**
    * **역할:** Rust(백엔드)와 웹(프론트엔드)을 하나의 가벼운 네이티브 앱으로 패키징합니다.
* **핵심 엔진 (Backend): Rust**
    * **역할:** macOS의 저수준(Native) API를 제어합니다. 시스템 전역 키보드 후킹, 권한 확인, 가상 키 입력 등 핵심 로직을 담당합니다.
* **설정 UI (Frontend): Web (HTML/CSS/JS)**
    * **역할:** Tauri 웹뷰(WebView)를 통해 '설정 창'이나 '권한 안내' 등 사용자에게 보여줄 화면을 구현합니다.
* **데이터베이스: SQLite**
    * **역할:** `bible.json` 데이터를 앱에 내장하여 빠르고 안정적인 오프라인 조회를 지원합니다.

## 4. 필요 자원 (Data Source)

* **원본 파일:** `bible.json` (사용자 제공)

## 5. 개발 진행 방식 (Implementation Plan)

### 1단계: 개발 환경 구축 (macOS)

1.  `xcode-select --install` 명령어로 **Xcode Command Line Tools** 설치
2.  **Homebrew** (패키지 관리자) 설치
3.  **Rust** (via `rustup`) 설치
4.  **Node.js** (via `brew`) 설치
5.  **Tauri CLI** (`brew install tauri-cli`) 설치
6.  **VS Code** (코드 에디터) 및 `rust-analyzer`, `Tauri` 확장(Extension) 설치

### 2단계: 데이터베이스 구축 (Preprocessing)

* **목표:** `bible.json`을 `bible.sqlite` DB 파일로 변환합니다.
* **구현:**
    1.  `bible.json`을 읽는 스크립트(Python/Rust/Node.js) 작성.
    2.  SQLite DB 파일(`bible.sqlite`)을 생성하고 `verses` 테이블(예: `book_abbrev`, `chapter`, `verse`, `text`)을 정의.
    3.  JSON 데이터를 `INSERT` 쿼리로 변환하여 DB에 모두 삽입.
    4.  완성된 `bible.sqlite` 파일을 Tauri 프로젝트 폴더에 리소스로 포함.

### 3단계: 핵심 엔진 개발 (Rust)

1.  **키보드 후킹 (Hooking):**
    * `core-graphics` 라이브러리의 `CGEvent.tapCreate` API를 사용하여 시스템 전역의 키보드 이벤트를 실시간으로 감지합니다.
2.  **'손쉬운 사용' 권한 처리:**
    * 이 앱은 사용자가 `시스템 설정 > 개인정보 보호 > 손쉬운 사용`에서 '허용'해야 작동합니다.
    * `AXIsProcessTrusted()` 같은 API로 현재 앱이 권한을 가졌는지 확인합니다.
    * 권한이 없을 경우, 키보드 후킹을 시도하지 않고 UI(JS) 측에 "권한 요청 안내" 신호를 보냅니다.
3.  **패턴 분석 로직:**
    * 입력된 키를 버퍼(Buffer)에 저장합니다.
    * 정규표현식(Regex)을 사용해 `#` + `(약어)` + `(숫자)` + `:` + `(숫자)` + `(Space/Enter)` 패턴을 감지합니다.
4.  **한글 약어 매핑 (Mapping):**
    * 사용자가 입력한 '한글 약어'(예: "창", "창세기")를 `bible.json`의 '영어 약어'(`"gn"`)로 변환하는 `HashMap` 로직을 구현합니다. (사용자가 제공할 목록 기반)
5.  **DB 조회:**
    * `rusqlite` 라이브러리를 사용해 `bible.sqlite` 파일에서 해당 구절을 `SELECT` 합니다.
6.  **텍스트 주입 (Injection):**
    * `enigo` 같은 가상 입력 라이브러리를 사용합니다.
    * **동작:**
        1.  입력된 약어(예: `#창 1:1 `)의 길이만큼 가상 `Backspace` 키 이벤트를 OS에 보냅니다.
        2.  DB에서 조회한 구절 텍스트를 가상 키보드 입력 이벤트로 OS에 보냅니다.

### 4단계: UI 개발 (Web)

1.  **권한 요청 화면 (최우선):**
    * 엔진(Rust)으로부터 "권한 없음" 신호를 받으면, "손쉬운 사용 권한이 필요합니다"라는 안내 창을 표시합니다.
    * '설정 열기' 버튼(사용자를 시스템 설정으로 이동)을 제공합니다.
2.  **설정 창 (메뉴 막대 아이콘 클릭 시):**
    * 기능 On/Off 토글 스위치
    * **트리거 문자:** `#` (기본값)
    * PC 시작 시 자동 실행 체크박스

### 5단계: 통합 및 패키징 (Tauri)

1.  Rust 엔진(Backend)과 Web UI(Frontend)를 Tauri API로 연결합니다.
2.  macOS 상단 메뉴 막대(Menu Bar) 아이콘 및 메뉴(설정, 종료)를 설정합니다.
3.  `cargo tauri build` 명령어로 최종 `.app` 및 `.dmg` 설치 파일을 생성합니다.
